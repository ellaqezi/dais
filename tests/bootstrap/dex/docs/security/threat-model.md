## System Overview

devexnet is a Python CLI tool that reads YAML configuration files, runs external validation
tools as subprocesses, and writes structured output to the filesystem. It does not make
network calls, does not store user data, and does not expose a web interface.

## Threat Catalog (STRIDE)

### T01 — Prompt Injection (CRITICAL)

**Category**: Injection
**Description**: devexnet processes loom-reed-light prompt files and DAIS agent prompts as
inputs. If user-supplied content from task files or spec files is passed unsanitized as a
system prompt or shell argument, an attacker can inject arbitrary instructions to the
downstream model or shell.

**Mitigations**:
- Prompt files are read as raw text only; never interpolated into shell commands
- User-supplied strings passed to `subprocess` must go through `src/validators.py`
  allow-list before use
- Path traversal: all file paths are resolved with `pathlib.Path.resolve()` and checked
  to remain within the project root before any read or write
- `yaml.safe_load()` only — never `yaml.load()` with an arbitrary Loader

**Residual risk**: LOW (with mitigations applied)

### T02 — Path Traversal (HIGH)

**Category**: Tampering
**Description**: `devexnet validate <path>` accepts a target directory from the user.
A crafted path (`../../etc/passwd`) could cause reads or writes outside the intended scope.

**Mitigation**: Resolve all paths with `pathlib.Path(user_input).resolve()`. Assert the
resolved path is a subdirectory of a safe root before any operation.

**Residual risk**: LOW (with mitigation)

### T03 — Dependency Supply Chain (MEDIUM)

**Category**: Tampering
**Description**: devexnet depends on pre-commit, detect-secrets, shellcheck, and PyYAML.
A compromised upstream package could introduce malicious behavior.

**Mitigation**:
- Pin all dependency versions in `pyproject.toml`
- Use `detect-secrets` in CI to prevent credential commits
- Review dependency updates via Dependabot or equivalent

**Residual risk**: MEDIUM (ongoing)

### T04 — Secrets in .env (MEDIUM)

**Category**: Information Disclosure
**Description**: `.env` files may contain model API keys or path configs. If committed,
these are permanently in git history.

**Mitigation**:
- `.env` is in `.gitignore` and `.secrets.baseline` covers its pattern
- `detect-secrets` hook blocks any commit containing high-entropy strings
- `.env.example` contains only non-sensitive placeholders

**Residual risk**: LOW (with mitigation)
